# Портфолио фотографа

Современный full-stack проект для сайта фотографа: публичное портфолио, страницы услуг и блога, форма обратной связи, API для контента и готовая Docker-инфраструктура для деплоя.

Стек:

- `frontend`: React + Vite
- `backend`: FastAPI + SQLAlchemy
- `database`: PostgreSQL
- `proxy`: Nginx
- `infra`: Docker Compose

## Что это за проект

Это сайт-портфолио для фотографа с готовым пользовательским интерфейсом и backend API.

Проект решает типовые задачи такого сайта:

- показать лучшие работы на главной и в отдельной галерее
- рассказать о фотографе и формате работы
- вывести список услуг
- публиковать заметки или истории со съёмок
- принимать заявки через форму обратной связи
- хранить и отдавать медиафайлы
- быстро развернуть проект через Docker Compose

Подходит как стартовая база для:

- персонального сайта фотографа
- landing page для записи на съёмку
- миграции со старого сайта с импортом медиа
- дальнейшего развития под CMS / админку / CRM-интеграции

## Основные возможности

### Публичная часть сайта

- главная страница с hero-блоком и подборкой работ
- отдельная страница портфолио с сеткой изображений
- страница "Обо мне" с описанием, портретом и счётчиками
- страница услуг
- страница блога
- страница контактов с формой заявки
- адаптивный интерфейс для десктопа и мобильных устройств

### Работа с изображениями

- хранение пользовательских загрузок в `UPLOAD_DIR`
- публикация медиа через `/media/...`
- lightbox-просмотр изображений на страницах
- fallback на демонстрационные изображения темы, если API ещё не заполнено
- импорт legacy-медиа из старого каталога

### Backend API

- `GET /api/health` — проверка, что API живо
- `GET /api/health/ready` — проверка готовности API и БД
- `GET /api/settings` — публичные настройки сайта
- `GET /api/collections` — портфолио с пагинацией
- `GET /api/blog` — блог с пагинацией
- `GET /api/services` — список услуг
- `POST /api/feedback` — отправка сообщения через форму контактов

### Контент и сидирование

- автоматическое создание таблиц при старте приложения
- начальное заполнение базовых данных
- импорт изображений и контента из legacy-структуры
- заполнение демо-контентом из theme assets, если проект пустой

### Инфраструктура

- запуск всего проекта одной командой через Docker Compose
- отдельный compose-оверлей для Traefik
- Nginx как внешний вход в приложение
- PostgreSQL в отдельном контейнере
- CI с backend- и frontend-тестами, а также проверкой compose-конфигурации

## Как работает сайт

Frontend запрашивает данные у FastAPI API:

- настройки сайта
- коллекции работ
- статьи блога
- услуги

Backend:

- читает настройки из БД
- отдает нормализованные URL изображений
- сохраняет сообщения из формы обратной связи
- при необходимости отправляет email-уведомления
- раздаёт загруженные медиа через mounted static directory

Если данных в API ещё нет, frontend показывает аккуратные fallback-данные и изображения темы, поэтому сайт не выглядит пустым даже на первом запуске.

## Модели данных

В проекте есть основные сущности:

- `SiteSettings` — имя фотографа, заголовки, bio, контакты, hero/about изображения, счётчики

## Быстрый старт через Docker

### 1. Подготовьте `.env`

Скопируйте шаблон:

- `cp env.example .env`

Минимально проверьте и задайте:

- `POSTGRES_PASSWORD`
- `DATABASE_URL`
- `CORS_ORIGINS`
- `HTTP_PORT`

При необходимости:

- `UPLOAD_DIR`
- `LEGACY_MEDIA_IMPORT_DIR`
- `STATIC_SEED_IMAGES_DIR`
- `SMTP_HOST`
- `SMTP_PORT`
- `SMTP_USER`
- `SMTP_PASSWORD`
- `FEEDBACK_FROM_EMAIL`
- `FEEDBACK_TO_EMAIL`

### 2. Запустите проект

```/dev/null/commands.sh#L1-1
docker compose -f docker-compose.yml -f docker-compose.http.yml up -d --build
```

### 3. Проверьте запуск

После старта должны открываться:

- главная страница
- `GET /api/health`
- `GET /api/health/ready`

## Переменные окружения

Все runtime-настройки вынесены в `.env`.

Основные группы:

- база данных: `POSTGRES_*`, `DATABASE_URL`
- сеть и доступ: `HTTP_PORT`, `CORS_ORIGINS`
- медиа: `UPLOAD_DIR`, `LEGACY_MEDIA_IMPORT_DIR`, `LEGACY_MEDIA_BIND`
- сидирование: `STATIC_SEED_IMAGES_DIR`, `FORCE_LEGACY_MEDIA_REIMPORT`
- email и обратная связь: `SMTP_*`, `FEEDBACK_*`
- TLS / домен: `TRAEFIK_DOMAIN`, `ACME_EMAIL`

## Импорт legacy-медиа

Если у вас есть старый архив изображений, проект умеет подтянуть его при старте.

Типовой сценарий:

- смонтировать старый каталог в контейнер
- указать `LEGACY_MEDIA_IMPORT_DIR`
- запустить приложение
- backend скопирует изображения в uploads и создаст базовые записи для коллекции, блога и части настроек сайта

Для повторного импорта:

- временно установить `FORCE_LEGACY_MEDIA_REIMPORT=true`
- выполнить запуск
- вернуть значение обратно в `false`

## Безопасность и деплой

- наружу публикуется только Nginx
- API не обязательно открывать напрямую на хост
- чувствительные данные задаются через `.env`
- для TLS доступен compose-вариант с `docker-compose.traefik.yml`

Запуск с Traefik:

```/dev/null/commands.sh#L1-1
docker compose -f docker-compose.yml -f docker-compose.traefik.yml up -d --build
```

Подробности по серверному развёртыванию: `deployment.md`.
